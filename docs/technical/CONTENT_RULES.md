# Zasady treści

## Nie wolno

- publikować fikcyjnych projektów,
- publikować roboczych opisów jako oferty,
- używać wygenerowanych zdjęć jako realnych zdjęć projektu,
- trzymać prywatnych PDF w `public`.

## Wolno

- używać placeholderów, gdy brakuje zdjęcia,
- trzymać referencje wizualne w docs,
- dodać projekt jako draft,
- opublikować tylko projekt z realnymi danymi.

## Projekt jest publiczny tylko gdy

```json
"status": "active"
```
