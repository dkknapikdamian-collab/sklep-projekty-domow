# Zasady projektu

Ten folder jest źródłem zasad projektowych i decyzji operacyjnych.

## Po co istnieje

Projekt będzie długo rozwijany. Część ustaleń może się zmieniać.

Nie wolno cicho nadpisywać wcześniejszych decyzji, jeśli nowa decyzja zmienia logikę działania systemu.

## Zasada konfliktu decyzji

Jeśli nowe ustalenie jest sprzeczne z wcześniejszym:

1. trzeba wskazać konflikt,
2. trzeba powiedzieć, co było ustalone wcześniej,
3. trzeba powiedzieć, co zmieniamy,
4. trzeba zapisać zmianę w decision logu,
5. dopiero potem wdrażać nową logikę.
