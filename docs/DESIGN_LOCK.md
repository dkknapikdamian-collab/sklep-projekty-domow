# DESIGN LOCK

Ta wersja kodu jest pierwszym działającym przeniesieniem zaakceptowanego kierunku wizualnego do aplikacji.

Zasady:
- nie zmieniać ciemnego grafitu,
- nie zmieniać złoto-beżowego CTA,
- karta projektu ma mieć prawy box zakupowy,
- projekt ma mieć duże wizualizacje,
- komponenty mają korzystać z jednego stylu,
- dane mają iść z `data/projects.ts`, a później z bazy.

Następny krok:
Po akceptacji UI podpinamy bazę i panel admina.
